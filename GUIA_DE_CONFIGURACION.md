# Stock Integral — Guía de configuración (10-15 minutos, una sola vez)

Tu app ahora tiene dos cosas nuevas:

1. **Se puede instalar en el celular** como si fuera una app de verdad (ícono en el escritorio, pantalla completa, funciona sin internet para consultar lo último visto).
2. **Sincronización automática** entre todos tus dispositivos, usando una base de datos gratuita en la nube (Supabase).

Sin el paso 2 hecho, la app funciona igual que antes (todo guardado solo en ese celular/PC). Con el paso 2, todo lo que cargues en cualquier dispositivo aparece automáticamente en los demás.

---

## Paso 1 — Crear tu base de datos gratuita en Supabase

1. Andá a **https://supabase.com** y creá una cuenta gratis (podés entrar con Google).
2. Creá un **New Project**. Elegí cualquier nombre (ej. "stock-integral") y una contraseña para la base de datos (guardala, aunque no la vas a necesitar para esta app).
3. Esperá 1-2 minutos a que el proyecto termine de crearse.
4. En el menú de la izquierda, andá a **SQL Editor** → **New query**, pegá esto y tocá **Run**:

```sql
create table movimientos (
  id bigint generated by default as identity primary key,
  tipo text not null,
  articulo text not null,
  cantidad integer not null,
  origen text not null,
  destino text not null,
  fecha date not null,
  obs text default '',
  created_at timestamptz default now()
);

alter table movimientos enable row level security;

create policy "acceso total anon"
on movimientos for all
to anon
using (true)
with check (true);

alter publication supabase_realtime add table movimientos;
```

Esto crea la tabla donde se guardan tus movimientos, y habilita que la app (que usa una clave pública "anon") pueda leer y escribir en ella, y que los cambios se avisen en tiempo real a los demás dispositivos.

> **Nota sobre seguridad:** esta configuración deja la tabla abierta a quien tenga tu URL y tu clave "anon" (que van dentro del código de la app). Como es un proyecto para uso personal/interno y no vas a compartir públicamente el link de la app, el riesgo práctico es bajo. Si en algún momento querés protegerlo con usuario y contraseña, avisame y lo agregamos (Supabase Auth).

5. Andá a **Project Settings** (ícono de tuerca) → **API**. Vas a ver dos datos que necesitás:
   - **Project URL** (algo como `https://xxxxxxxx.supabase.co`)
   - **anon public key** (una clave larga que empieza con `eyJ...`)

Guardá esos dos datos, los vas a pegar en la app en el paso 3.

---

## Paso 2 — Publicar la app para poder instalarla en el celular

Un service worker (lo que permite instalar la app y usarla offline) necesita que la app esté en una dirección https, no simplemente abierta como archivo local. La forma más rápida y gratuita:

### Opción recomendada: Netlify Drop (sin cuenta, sin instalar nada)

1. Andá a **https://app.netlify.com/drop**
2. Arrastrá la carpeta completa `stockapp` (con `index.html`, `manifest.json`, `sw.js`, los íconos) a esa página.
3. En segundos te da un link público (ej. `https://algo-random.netlify.app`). Ese es el link de tu app.

*(Alternativas igual de válidas: GitHub Pages o Vercel, si ya las usás.)*

---

## Paso 3 — Conectar la app a tu base de datos

1. Abrí el link de tu app (del paso 2) desde el celular o la compu.
2. Tocá el botón **"Sin sincronizar"** arriba a la derecha (o "Sincronización" en el menú lateral).
3. Pegá la **Project URL** y la **anon public key** que guardaste en el Paso 1.
4. Tocá **Guardar y conectar**. Si todo salió bien, el botón de arriba va a decir **"Sincronizado"**.

Repetí el paso 3 en cada dispositivo desde el que quieras cargar datos (con la misma URL y key). A partir de ahí, todo se sincroniza solo.

---

## Paso 4 — Instalar la app en el celular

- **Android (Chrome):** abrí el link → menú (⋮) → **"Instalar app"** o **"Agregar a pantalla de inicio"**.
- **iPhone (Safari):** abrí el link → botón de compartir (□↑) → **"Agregar a pantalla de inicio"**.

Va a quedar con su ícono propio, y se abre en pantalla completa como una app nativa.

---

## Preguntas frecuentes

**¿Qué pasa si pierdo internet?** Podés seguir viendo los últimos movimientos cargados (quedan en caché). Para cargar movimientos nuevos que se sincronicen, necesitás conexión.

**¿Puedo dejar de sincronizar y volver a modo local?** Sí, con el botón "Desconectar" dentro de Ajustes → Sincronización. Los datos que ya estén en la nube no se borran.

**¿Puedo compartir esto con empleados en cada local para que carguen ellos?** Sí es posible, pero conviene agregar un login simple para que cada uno tenga su usuario. Si más adelante lo necesitás, decime y lo sumamos.
